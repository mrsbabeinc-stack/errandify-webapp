Severity: 🔴 CRITICAL
Location: /backend/database/schema.sql
Issue: Code references 'confirmation_expires_at' column but it doesn't exist

Error Message:
  HTTP 500: "Failed to accept bid"

Root Cause:
  File: /backend/src/routes/bids.ts (line 221)
  Code: UPDATE errands SET confirmation_expires_at = NOW() + INTERVAL '24 hours'
  Problem: Column 'confirmation_expires_at' doesn't exist in errands table

Solution:
  Create database migration to add missing column:
  ALTER TABLE errands ADD COLUMN confirmation_expires_at TIMESTAMP;

Time to Fix: 2 minutes

BUG #2: Assignment Record Not Created on Bid Accept
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Severity: 🔴 CRITICAL (Secondary Issue)
Location: /backend/src/routes/bids.ts
Issue: When bid is accepted, no record created in errand_assignments table

Error Message:
  "You are not assigned to this task" (when doer tries to start)

Root Cause:
  The POST /:id/accept handler updates the errands table but:
  - Does NOT INSERT into errand_assignments
  - Without this, task_start endpoint rejects the request

Solution:
  Add INSERT statement after bid acceptance:
  INSERT INTO errand_assignments (errand_id, doer_id, status, created_at)
  VALUES (...) ON CONFLICT DO UPDATE

Time to Fix: 3 minutes
