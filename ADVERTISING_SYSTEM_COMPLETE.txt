================================================================================
STRIPE PAYMENT INTEGRATION
================================================================================

Campaign Approval Flow:
1. Admin calls POST /api/admin/advertising/approve
2. Backend creates Stripe charge for full budget amount (SGD)
3. If charge succeeds:
   - Store charge ID in campaigns table
   - Create payment hold in PaymentHoldsStatus (status='held')
   - Update campaign status to 'approved'
   - Create ad_schedules for start_date and end_date
   - Send approval email to company with charge amount
4. If charge fails:
   - Auto-reject campaign
   - Send rejection email with error message
   - No payment hold created

Features:
- Idempotent: Safe to retry (same charge ID used)
- Atomic: Charge and database update together
- Secure: All Stripe calls server-side only
- Fallback: Test charges for development

CAMPAIGN LIFECYCLE
================================================================================

Status Flow:
  Draft
    ↓ (Company edits & submits)
  Submitted (Admin receives email)
    ↓ (Admin Action)
    ├→ Approved [Stripe charge here] → Live [Auto at start_date]
    │                                    ├→ Pause → Paused
    │                                    │            └→ Resume → Live
    │                                    └→ End [Auto at end_date] → Expired
    │
    └→ Rejected [Auto on Stripe failure]
        ↓ (Company edits & resubmits)
      Draft (cycle repeats)

Payment Flow:
- Draft:        No charge
- Submitted:    No charge (awaiting review)
- Approved:     CHARGE NOW for full budget
- Live:         Daily spending tracked via mock data
- Expired:      Payment hold released, final metrics calculated

================================================================================
TROUBLESHOOTING
================================================================================

Problem: Campaign not transitioning to live
Solution:
1. Check ad_schedules table: SELECT * FROM ad_schedules WHERE campaign_id = X
2. Verify scheduled_date is in the past
3. Check cron logs for errors
4. Manual fix: UPDATE campaigns SET status = 'live' WHERE id = X

Problem: Stripe charge fails on approval
Solution:
1. Check company has stripe_customer_id
2. Verify payment method in Stripe account
3. Check card has sufficient funds
4. Campaign auto-rejected with error message

Problem: Performance data not updating
Solution:
1. Check campaign has status='live'
2. Wait until 1 AM or trigger cron manually
3. Check server logs for: [AD_SCHEDULER] Generated mock data
4. Verify campaign_performance table has entries

Problem: Notifications not sending
Solution:
1. Check email service configured in .env
2. Verify owner email in companies table
3. Check notifications table has records
4. Review server logs for email errors
