## PART 1: WHO ARE VULNERABLE USERS?

### Target Groups (Singapore Context)

```
1. ELDERLY (60+)
   • May struggle with technology
   • Often lonely, susceptible to scams
   • Fixed income, can't afford fraud loss
   • Physical limitations (vision, hearing, mobility)
   • May struggle reading reviews
   • Need large fonts, simple language

2. MIGRANT WORKERS & FOREIGN DOMESTIC HELPERS
   • Language barriers (not fluent in English)
   • Unfamiliar with Singapore culture/laws
   • Often isolated, few friends
   • May fear authorities (visa concerns)
   • Vulnerable to exploitation
   • Need multi-language support
   • Need cultural context

3. PERSONS WITH DISABILITIES
   • Physical: Mobility, vision, hearing
   • Cognitive: Difficulty with complex tasks
   • Mental health: Anxiety, depression
   • Need accessible UI (WCAG 2.1 AAA not just AA)
   • May need assisted technology
   • Need clear, simple language

4. LOW-INCOME INDIVIDUALS
   • Can't afford mistakes/fraud
   • May not have credit cards (use e-wallet)
   • Limited device/internet access
   • May struggle with complex interfaces
   • Need clear pricing, no hidden fees
   • May distrust "too good to be true"

5. ABUSE SURVIVORS & AT-RISK INDIVIDUALS
   • Trauma responses (anxiety, hypervigilance)
   • May avoid confrontation (won't rate honestly)
   • May fear retaliation
   • Need safe, private environment
   • Need clear escalation paths
   • Need trauma-informed language

6. VICTIMS OF TRAFFICKING & MODERN SLAVERY
   • May not have control over finances
   • May fear reporting
   • Handler may monitor communications
   • Need covert safety features
   • Need crisis support resources
   • May need to hide activity

## PART 3: AI ENHANCEMENTS FOR VULNERABLE USERS

### AI Feature 1: Automatic Fair Rating Detection

```
PROBLEM: Vulnerable users might rate unfairly because:
- Fear of retaliation ("If I rate low, they'll get revenge")
- Pressure from handler ("Give good reviews or else")
- Low confidence ("Maybe it's my fault, not theirs")
- Language barriers ("Maybe I misunderstood")

SOLUTION: AI Fair Rating Assistant

When vulnerable user rates 1-2 stars, AI checks:
1. Is this rating consistent with chat history?
   - If chat is positive but rating is negative: Flag as inconsistent
   - Example: "Chat showed 'Great job!' but rating is 1 star - are you sure?"

2. Is there evidence of coercion in chat?
   - Detected patterns: "You MUST give good review"
   - AI flags: "This sounds like pressure - are you OK?"

3. Is this rating supported by job details?
   - Job: "Clean house" → Rating 1 star "Didn't clean"
   - Consistent? Yes, flag is OK
   - Job: "Clean house" → Rating 1 star "I was scared"
   - Inconsistent? AI offers support resources

IMPLEMENTATION:

When submitting low rating:
AI asks: "You rated 1 star - you can tell us why, if you want to"

[Textarea: "Why did you rate 1 star?"]

Then AI analyzes response:
- "They were rude" → Valid low rating, approve
- "I'm scared" → Offer support resources
- "Handler told me to rate low" → Flag for safety team, offer help
- "I don't know, maybe it's my fault" → Coaching: "It's OK to have standards"

Result: Vulnerable users give HONEST ratings, not coerced ones
```

### AI Feature 2: Protective Language Detection

```
GOAL: Detect and block language that targets vulnerable users

PATTERNS DETECTED:

1. Age-based discrimination
   ✗ "Old people shouldn't use technology"
   ✓ "I had trouble explaining, could be clearer instructions"

2. Language-based discrimination
   ✗ "Foreign workers are always incompetent"
   ✓ "Communication was difficult, consider hiring bilingual staff"

3. Disability-based discrimination
   ✗ "Disabled people can't do this job"
   ✓ "The stairs made it difficult for them, wheelchair ramp needed"

4. Coercion signals
   ✗ "You MUST give good reviews or lose your job"
   ✗ "If you don't rate 5 stars, I'll report you"
   ✓ "I appreciated your professionalism"

AI ACTION:
- Automatically block discriminatory language
- Offer rewrite suggestion: "Try this instead..."
- Log for safety investigation
- Notify user: "This language isn't allowed - here's why..."
- No penalty for accidental insensitivity (offer education)
- Penalty only for intentional discrimination
```

### AI Feature 3: Protective Moderation for Vulnerable Users

```
CURRENT SYSTEM: One-size-fits-all moderation

IMPROVED SYSTEM: Vulnerable-aware moderation

When flagging review:
1. Check if reviewer is vulnerable (elderly, migrant, disabled, low-income)
2. If YES: Apply protective moderation rules
3. If NO: Apply standard moderation rules

EXAMPLE 1: Elderly Migrant Reviews Low
Review: "Worker not good, not understand English"
Standard moderation: "Might be age-based discrimination" → Hold for review
Vulnerable-aware: "Could be legitimate language barrier" + "Protect reviewer"
   → Auto-approve (language barrier is valid reason)
   → Monitor for harassment of this vulnerable reviewer
   → If doer harasses: Immediate suspension

EXAMPLE 2: Abuse Survivor Gives Negative Review
Review: "Felt unsafe, worker made me uncomfortable"
Standard: "Vague, needs more detail" → Hold
Vulnerable-aware: "Trauma response legitimate" + "Protect reviewer safety"
   → Auto-approve
   → Don't ask for specifics (re-traumatizing)
   → Flag doer for safety investigation
   → Offer trauma support to reviewer

EXAMPLE 3: Trafficked Worker Can't Give Honest Review
Review: "5 stars, very good" (but chat history shows distress)
Standard: "Seems fine" → Approve
Vulnerable-aware: "Pattern detected" + "Coercion likely"
   → Don't publicly approve (protects reviewer)
   → Quietly flag for trafficking investigation
   → Show safety resources to reviewer
   → Don't contact doer (might endanger trafficked worker)
```

```

   ✓ Action: Investigate, ban if targeting confirmed

2. Vulnerable User Being Exploited
   ✓ Migrant worker rates 5 stars consistently (unusual for job quality)
   ✓ Chat shows pressure to rate high
   ✓ Action: Check-in with worker, investigate handler

3. Vulnerable User Showing Distress Signals
   ✓ Language suggests fear, coercion, trauma
   ✓ Sudden behavior change (started rating fearfully)
   ✓ Action: Proactively reach out with support resources

4. Abuse/Trafficking Indicators
   ✓ Worker can't meet their own needs (handler involved)
   ✓ Communication monitored/restricted
   ✓ Isolation signals (no friends, no autonomy)
   ✓ Action: Contact relevant NGOs, activate safety protocols

IMPLEMENTATION:
```

When vulnerable user submits review:
1. AI analyzes for distress signals
2. If found: Auto-approve review (protects them)
3. Show support resources
4. Flag for safety team review (not for disciplining user)
5. Monitor future activity for escalation
6. If escalation detected: Reach out proactively

```

## PART 7: DATABASE SCHEMA

```sql
-- Extend users table
ALTER TABLE users ADD COLUMN (
  is_vulnerable BOOLEAN DEFAULT false,
  preferred_language VARCHAR(20) DEFAULT 'en',
  accessibility_needs JSONB, -- {vision, hearing, motor, cognitive}
  support_needs JSONB -- {trauma, trafficking, low_income, etc}
);

-- New table: Vulnerable user protections
CREATE TABLE vulnerable_user_flags (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  flag_type VARCHAR(50), -- 'elderly', 'migrant', 'disabled', 'low_income', 'trauma', 'trafficking'
  added_by VARCHAR(50), -- 'auto_detected', 'self_declared', 'admin'
  details JSONB, -- {language: 'Tamil', support_contact: '1800-SAFE'}
  created_at TIMESTAMP DEFAULT NOW(),
  resolved_at TIMESTAMP
);

-- New table: Review safety flags
CREATE TABLE review_safety_flags (
  id SERIAL PRIMARY KEY,
  review_id INTEGER REFERENCES user_reviews(id),
  flag_type VARCHAR(50), -- 'fair_rating_inconsistency', 'coercion_detected', 'distress_signal'
  severity VARCHAR(20), -- 'low', 'medium', 'high', 'critical'
  ai_analysis JSONB, -- {signals: [...], recommendations: [...]}
  action_taken VARCHAR(50), -- 'auto_approved', 'held_for_review', 'escalated'
  created_at TIMESTAMP DEFAULT NOW(),
  resolved_at TIMESTAMP
);

-- New table: Safety team alerts
CREATE TABLE safety_alerts (
  id SERIAL PRIMARY KEY,
  alert_type VARCHAR(50), -- 'targeting', 'exploitation', 'trafficking', 'distress'
  user_id INTEGER REFERENCES users(id),
  related_doer_id INTEGER REFERENCES users(id),
  summary TEXT,
  evidence JSONB, -- {pattern: [...], timeline: [...]}
  priority VARCHAR(20), -- 'low', 'medium', 'high', 'critical'
  action_suggested TEXT,
  status VARCHAR(20) DEFAULT 'open', -- 'open', 'investigating', 'resolved', 'escalated_to_ngo'
  created_at TIMESTAMP DEFAULT NOW(),
  resolved_at TIMESTAMP,
  resolved_by_admin_id INTEGER REFERENCES users(id)
);
```
